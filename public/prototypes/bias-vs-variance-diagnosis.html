Bias vs. Variance
Diagnosis

This page is not just about seeing underfit or overfit curves. It teaches how to read the error gaps: compare human-level performance, training error, training-dev error, dev error, and test error, then decide whether the real bottleneck is bias, variance, data mismatch, or overfitting to the dev set.

Key concepts
What to watch for
Bias If human-level → training gap is the largest, the model still cannot fit the training distribution well.
Variance If training → training-dev or training → dev gap grows, the model does not generalize well to unseen examples.
Mismatch If training-dev is much better than dev, the issue is not only generalization, but also distribution shift.

Core logic
How to interpret the numbers
Bias diagnosis
If the gap from human-level error to training error is the largest, the model still cannot fit the training set well.

Variance diagnosis
If the training → training-dev or training → dev gap is large, the model is not generalizing well.

Mismatch diagnosis
If training-dev stays much better than dev, distribution shift is a major issue.

Dev overfitting
If test becomes worse than dev, you may be tuning too hard to the dev set.

Takeaway
Teaching summary
Diagnosis is not about one pretty curve. It is about comparing where each gap appears. Human → train tells you about bias. Train → training-dev tells you about variance. Training-dev → dev tells you about mismatch. Dev → test tells you whether you are starting to overfit the dev set.
